# /board — Boardroom Perspectives Analysis

Run a boardroom-style multi-perspective analysis on the given topic.

Usage: /board [topic or decision]

Instructions:
1. Analyze the topic from 5 perspectives. For each, give a 2–4
   sentence take — what this role would focus on, what they'd push
   back on, and what they'd want to see next:
   - **CEO / Strategic** — long-term direction, competitive position,
     brand
   - **CFO / Financial** — cost, cash flow, ROI, risk exposure
   - **CMO / Marketing** — customer acquisition, positioning, message
   - **COO / Operational** — execution feasibility, capacity, process
   - **Customer / End-user** — will they actually want this, pay
     for it, notice it
2. Flag any conflicts between perspectives.
3. Synthesize: give a weighted recommendation with the top 2–3
   reasons, and one clear next action.

Saving:
- Append the synthesis to `10-stellaris-ridge/_decisions.md`
- Format: `## [YYYY-MM-DD] — [Topic]` followed by the synthesis
  and next action
- Ask Carl to confirm before saving if the decision has
  significant implications (spending, new hires, client
  commitments, strategic pivots).
